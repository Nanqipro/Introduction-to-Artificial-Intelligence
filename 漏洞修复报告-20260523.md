## 二、逐项修复详情

---

### 原始漏洞 — 越权访问章节接口（已于上次修复，本次确认）

**漏洞描述：** 攻击者无需任何身份验证，可直接调用 `POST /api/chapters` 创建章节、`DELETE /api/chapters/{id}` 删除任意章节。

**修复方案：**
- 将章节写操作（创建、更新、删除）从公开的 `ChapterController` 迁移至新建的 `AdminChapterController`，路径前缀改为 `/api/admin/chapters`
- `/api/admin/**` 路径受 `LoginInterceptor`（验证 JWT）和 `AdminInterceptor`（验证 role=admin）双重拦截器保护
- `ChapterController`（`/api/chapters`）保留只读 GET 接口

**修复后接口对照：**

| 操作 | 修复前 | 修复后 |
|------|--------|--------|
| 创建章节 | `POST /api/chapters`（无需认证） | `POST /api/admin/chapters`（需管理员 JWT） |
| 删除章节 | `DELETE /api/chapters/{id}`（无需认证） | `DELETE /api/admin/chapters/{id}`（需管理员 JWT） |
| 更新章节 | 同上 | `PUT /api/admin/chapters/{id}`（需管理员 JWT） |

### VULN-01 — 管理员密码硬编码在前端 JS（🔴 严重）

**漏洞描述：** 路由守卫中的权限拦截弹窗将管理员账号 `goodlabAdmin` 和密码 `goodlabPwd` 以明文写在前端 JS 代码中，任何用户打开浏览器开发者工具即可获取。

**修复文件：** `client/src/router/index.js`

```diff
- alert('访问被拒绝：只有管理员才能访问此功能\n\n管理员账号：goodlabAdmin\n管理员密码：goodlabPwd')
+ alert('访问被拒绝：您没有管理员权限')
```

**修复效果：** 前端代码不再包含任何凭证信息，攻击者无法通过源码获取管理员账号密码。

### VULN-02 — 修改密码无需验证旧密码（🟠 高危）

**漏洞描述：** `PATCH /api/user/updatePwd` 接口仅在请求中携带 `oldPwd` 参数时才验证旧密码，若不携带则直接跳过验证。前端调用时也刻意不传 `oldPwd`，导致任何持有有效 JWT Token 的攻击者可在不知道原密码的情况下修改任意账号密码，永久劫持账号。

**修复文件：** 共 3 个文件

**① `server/src/main/java/com/goodlab/server/controller/UsersController.java`**

```diff
- // 如果提供了旧密码，则进行验证；否则跳过
- if (!StringUtils.isEmpty(oldPwd)) {
-     User user = userService.findByUserName(username);
-     if(!Md5Util.getMD5String(oldPwd).equals(user.getPassword())){
-         return ApiResponse.error("原密码错误");
-     }
- }
+ // 强制验证旧密码，防止持有 token 的攻击者直接改密劫持账号
+ if(!StringUtils.hasText(oldPwd)){
+     return ApiResponse.error("原密码不能为空");
+ }
+ User user = userService.findByUserName(username);
+ if(!Md5Util.getMD5String(oldPwd).equals(user.getPassword())){
+     return ApiResponse.error("原密码错误");
+ }
```

**② `client/src/services/api.js`**

```diff
  updatePassword: (passwordData) => {
-   // 仅提交新密码与确认密码
    const requestData = {
+     oldPwd: passwordData.oldPassword,
      newPwd: passwordData.newPassword,
      confirmPwd: passwordData.confirmPassword
    }
```

**③ `client/src/components/profile/UserInfo.vue`**

- 新增"原密码"输入框（`v-model="passwordData.oldPassword"`）
- `passwordData` reactive 对象新增 `oldPassword` 字段
- `formRules` 新增 `oldPassword` 验证规则：修改密码时原密码必填
- `handleSave` 中 `passwordChange` 对象加入 `oldPassword` 字段

**修复效果：** 修改密码时强制要求输入原密码，即使 Token 泄露，攻击者也无法在不知道当前密码的情况下修改密码。

**修复效果：** `GET /api/quiz/results/{chapterId}` 现在必须携带有效 JWT Token 才能访问，匿名请求返回 401。

### VULN-04 — JWT 默认密钥泄露风险（🟡 中危）

**漏洞描述：** `application.properties` 中 JWT 密钥使用了写死的默认值 `GoodLab-AI-Platform-Secret-Key-2026-Production-Use-Only`。若服务器未配置 `JWT_SECRET` 环境变量，任何获取过源码的人均可用此密钥伪造任意用户（含管理员）的 JWT Token，绕过全部认证。

**修复文件：** 共 4 个文件

**① `server/src/main/resources/application.properties`**

```diff
- jwt.secret=${JWT_SECRET:GoodLab-AI-Platform-Secret-Key-2026-Production-Use-Only}
+ # 必须通过环境变量 JWT_SECRET 设置，启动时若检测到使用占位符将拒绝启动
+ jwt.secret=${JWT_SECRET:INSECURE_PLACEHOLDER_MUST_SET_JWT_SECRET_ENV_VAR}
```

**② `server/src/main/java/com/goodlab/server/utils/JwtUtil.java`**

```diff
  @PostConstruct
  public void init() {
+     if (secretValue == null || secretValue.contains("INSECURE_PLACEHOLDER")) {
+         throw new IllegalStateException(
+             "JWT密钥未配置：请设置环境变量 JWT_SECRET（建议使用 openssl rand -base64 64 生成）");
+     }
      KEY = secretValue;
  }
```

**③ 新建 `server/.env`**（已加入 `.gitignore`，不进入版本库）

```
JWT_SECRET="<由 openssl rand -base64 64 随机生成的 88 位强密钥>"
```

**④ `start-backend.sh`**

```diff
- # JWT 签名密钥（生产环境，勿泄露）
- export JWT_SECRET="..."
+ # 从 server/.env 加载敏感环境变量，该文件不进入版本库
+ ENV_FILE="$(dirname "$0")/server/.env"
+ if [ -f "$ENV_FILE" ]; then
+     set -a
+     source "$ENV_FILE"
+     set +a
+ else
+     echo "✗ 缺少 server/.env 文件，请创建并配置 JWT_SECRET"
+     exit 1
+ fi
```

**修复效果：**
- 密钥从源码中彻底剥离，存储于不进入版本库的 `.env` 文件
- 若未配置密钥，应用启动时直接抛出异常拒绝启动，避免以不安全状态运行
- 密钥替换为随机生成的强密钥（88 位 Base64，熵值充足）

### VULN-05 — 密码重置强度策略不一致（🟡 中危）

**漏洞描述：** `POST /api/password-reset/reset` 接口的密码验证仅要求长度 ≥6 位，而其他修改密码接口均要求 ≥8 位且至少包含两种字符类型。攻击者可借此将账号密码降级为 `123456` 等弱密码。

**修复文件：** `server/src/main/java/com/goodlab/server/controller/PasswordResetController.java`

```diff
- if (newPassword == null || newPassword.length() < 6) {
-     return Result.error("密码长度不能少于6位");
- }
+ if (newPassword == null || newPassword.length() < 8) {
+     return Result.error("密码长度不能少于8位");
+ }
+ boolean hasNumber = newPassword.matches(".*\\d.*");
+ boolean hasLetter = newPassword.matches(".*[a-zA-Z].*");
+ boolean hasSpecial = newPassword.matches(".*[!@#$%^&*()_+\\-=\\[\\]{};':\"\\\\|,.<>\\/?@].*");
+ int typeCount = (hasNumber ? 1 : 0) + (hasLetter ? 1 : 0) + (hasSpecial ? 1 : 0);
+ if (typeCount < 2) {
+     return Result.error("密码必须包含数字、字母、特殊字符中的至少两种");
+ }
```

**修复效果：** 密码重置接口与 `updatePwd`、`first-login-password-change` 接口执行完全相同的密码强度策略，消除策略不一致漏洞。

### VULN-06 — 章节路径排除过宽（🟡 中危）

**漏洞描述：** `WebConfig.java` 使用通配符 `/api/chapters/**` 将 `/api/chapters/` 下所有路径排除出认证拦截。虽当前 `ChapterController` 仅有 GET 方法，但未来若开发者在此路径下新增写接口，会自动绕过认证，重现原始越权漏洞。

**修复文件：** `server/src/main/java/com/goodlab/server/config/WebConfig.java`

```diff
- "/api/chapters/**",
+ // 章节只读公开接口精确枚举，避免通配符意外暴露写接口
+ "/api/chapters", "/api/chapters/all",
+ "/api/chapters/health", "/api/chapters/*",
```

**修复效果：** 仅精确放行当前已知的只读端点，任何新增于 `/api/chapters/` 下的写接口默认需要认证，从根本上消除通配符带来的隐患。
